## 1. Business Requirements (Бизнес-требования)

### 1.1 Background and Strategic Opportunity

Автор документа (Artem Konuchov) пишет много SaaS-продуктов, которые в своей базовой инфраструктуре одинаковые: auth, организации/проекты, биллинг, базовые UI-компоненты, CI, dev-окружение. Основной инструмент разработки — Claude Code + собственные скиллы и плагины (включая FASTSAAS-плагин, `/apz:…`). Каждый новый SaaS начинается с повторного сбора этой базы — это трата времени, и раз от раза база получается слегка разной, без единой канонической версии.

**FASTSAAS — open SaaS starter-kit**, отражающий то, как автор хочет, чтобы стартовал любой новый SaaS-проект:

- одна зафиксированная архитектура (FastAPI + Next.js, Actor-Centric);
- готовый набор модулей, из которого собирается конкретный проект под задачу;
- кодовая база, максимально дружелюбная к Claude Code / Cursor / Windsurf (Vibe Coding).

**Ключевой принцип — модульность.** Kit состоит из минимального core-модуля и набора опциональных модулей (billing, FinOps, MCP-server, background-jobs, AI Command Bar и др.). Новый проект включает только нужные модули; неиспользуемое не тянется балластом.

**Почему такая база актуальна сейчас (2026):** Next.js 15 (App Router), shadcn/ui, FastAPI + SQLModel, MCP и Stripe Checkout стабилизировались и стали индустриальным стандартом. Одновременно Claude Code достиг уровня, когда AI-агент может закрывать целые feature-циклы — при условии, что кодовая база под это спроектирована (явные конвенции, shared types, `.cursorrules`, `CLAUDE.md`, структура под предсказуемые meta-команды).

**APZ-плагин как часть Kit.** Автор использует собственный набор Claude-Code-команд (`/apz:req:*`, `/apz:build:*`, `/apz:ship:*`) для requirements, workflow и релизов. FASTSAAS включает **универсальную версию APZ-плагина**, пригодную не только для FASTSAAS-проектов, — чтобы методология и инструменты приходили вместе со стартер-kit.

### 1.2 Business Opportunity

FASTSAAS — стартовый kit для начала нового SaaS-проекта за 1–2 дня вместо 1–2 месяцев. Базовое ядро даёт auth + организации + проекты + роли + базовый UI, а опциональные модули подключаются по надобности (billing, MCP-server, FinOps, background-jobs, AI Command Bar). Kit поставляется с APZ-плагином (универсальным) и конвенциями для Claude Code.

**Для кого создаётся:**

- **Первичный пользователь:** автор документа — для собственных SaaS-проектов. Это главная целевая аудитория; все решения принимаются в её интересах.
- **Вторичная аудитория (если Kit будет открыт):** solo-founders и маленькие команды (1–3 человека), которые работают в сходной парадигме — Claude Code / Cursor как основной инструмент, FastAPI + Next.js как стек, Actor-Centric / multi-tenant модель как подход.

*Assumption:* модель распространения — opensource permissive (MIT/Apache-2.0), но это не обязательное условие успеха. Если Kit останется внутренним, BO-5 (opensource-трекшен) снимается. TBD: подтвердить.

### 1.4 Customer or Market Needs

Типичные болевые точки автора (и похожих solo-developer'ов), которые FASTSAAS должен снимать:

1. **«Я снова собираю auth + orgs + projects с нуля»** — базовая инфраструктура повторяется от SaaS к SaaS, каждый раз чуть по-своему.
2. **«Я не знаю, что делал мой AI-агент»** — при запуске AI-flows теряется видимость: какая операция инициирована человеком, а какая — агентом от его имени.
3. **«Бэкенд и фронтенд расходятся»** — валидация, типы и модели дублируются; ошибки всплывают в рантайме, а не на этапе написания.
4. **«Claude Code плохо ориентируется в моём коде»** — кодовая база не структурирована под Vibe Coding: нет `.cursorrules` и `CLAUDE.md`, непонятные конвенции, разбросанные файлы, AI тратит много попыток.
5. **«Нет быстрого способа дать AI-агенту управлять системой»** — интеграция MCP пишется руками для каждого проекта.
6. **«Нельзя просто выбросить лишнее»** — стартер-kit тянет всю инфраструктуру (billing, jobs, FinOps и т. д.), даже если конкретному проекту она не нужна; удалять вручную долго и рискованно.
7. **«Мои Claude-Code-скиллы живут отдельно от проектов»** — APZ-плагин и другие скиллы приходится подключать к каждому проекту и адаптировать.

## 2. Vision of the Solution (Видение решения)

### 2.1 Vision Statement

**FOR** автора (и solo-developer'ов, работающих в сходной парадигме Claude Code + FastAPI + Next.js)
**WHO** начинает новые SaaS-проекты часто и хочет иметь единую, уже «свою», инфраструктуру и методологию на старте
**THE** FASTSAAS **IS A** модульный fullstack-kit (FastAPI + Next.js) с Actor-Centric ядром и универсальным APZ-плагином
**THAT** даёт за 1–2 дня локально работающий SaaS-MVP с auth, организациями, проектами, подпиской и опциональными модулями (MCP, FinOps, background-jobs и др.), которые включаются только при необходимости
**UNLIKE** generic SaaS-стартер-китов (Next.js SaaS, Supabase starters, FastAPI full-stack), которые монолитны, построены вокруг «одного человека-пользователя» и игнорируют AI-агентов как первоклассных акторов
**OUR PRODUCT** модульный, различает Human/Agent с `intent_hash` в audit trail, приходит с универсальным APZ-плагином для Claude Code, и структурирует код так, чтобы AI-агент мог расширять его надёжно через meta-команды.

### 2.2 Major Features

Фичи сгруппированы по модулям: **CORE** — базовое ядро, всегда включено; **OPT** — опциональные модули, подключаются по надобности.

### 2.3 Assumptions and Dependencies

**Допущения:**

- **AS-1:** Целевой разработчик знаком с TypeScript и Python на уровне уверенного пользователя; не требуется объяснять базовые концепции этих языков.
- **AS-2:** Основной инструмент разработки — Claude Code (допускаются Cursor / Windsurf в совместимой парадигме).
- **AS-3:** Модель распространения Kit — permissive opensource (MIT/Apache-2.0); допустимо остаться личным/внутренним. *TBD: подтвердить.*
- **AS-4:** При использовании optional-модулей пользователь имеет собственные аккаунты в соответствующих сервисах (Stripe — для FE-OPT-BILLING; LiteLLM/OpenAI/Anthropic — для FE-OPT-FINOPS; Trigger.dev — для FE-OPT-JOBS; Sentry — для FE-CORE-12).
- **AS-5:** Язык интерфейса Kit — английский; локализация — out of scope для v1.0.
- **AS-6:** PostgreSQL (14+) — единственная поддерживаемая СУБД в v1.0.
- **AS-7:** Core-модуль не зависит ни от одного optional-модуля — проект, собранный без optional, собирается, запускается и проходит CI.

**Зависимости (core):**

- **DE-1:** Next.js 15+ (App Router) — стабильность API.
- **DE-2:** FastAPI + SQLModel (Pydantic v2 + SQLAlchemy 2.x).
- **DE-3:** shadcn/ui (Radix UI) — копипаст-модель остаётся актуальной.
- **DE-4:** TanStack Query (React Query) v5+.
- **DE-5:** React Hook Form + Zod.
- **DE-6:** Alembic для миграций БД.
- **DE-7:** Docker + docker-compose.
- **DE-8:** Playwright (baseline).
- **DE-9:** Sentry (frontend SDK + backend SDK для Python/Node).
- **DE-10:** Claude Code + APZ-плагин (universal).

**Зависимости (optional, только при активации соответствующего модуля):**

- **DE-OPT-1:** Stripe (Checkout + Subscriptions API) — для FE-OPT-BILLING.
- **DE-OPT-2:** LiteLLM Gateway (self-host или managed) — для FE-OPT-FINOPS.
- **DE-OPT-3:** Trigger.dev (v3+ SDK) — для FE-OPT-JOBS.
- **DE-OPT-4:** MCP-спецификация (текущая версия, совместимость с Claude Desktop / Claude Code) — для FE-OPT-MCP и FE-OPT-CMDBAR.

## 3. Scope and Limitations (Границы и ограничения)

### 3.1 Scope of Initial Release (v1.0)

**Core (обязательно в v1.0):** все FE-CORE-1 … FE-CORE-13. Это минимальный набор, без которого Kit не выполняет BO-1 (старт MVP за ≤5 дней) и BO-4 (модульность).

- **FE-CORE-1..3 (Actor-Centric ядро):** ключевое отличие Kit.
- **FE-CORE-4 (Auth):** без регистрации Kit непригоден.
- **FE-CORE-5..6 (Design System + Layouts):** базовый UI.
- **FE-CORE-7 (Dev env):** `make dev` — минимальная DX-планка.
- **FE-CORE-8..9 (Vibe Coding DX + meta-commands):** реализация BO-2.
- **FE-CORE-10 (Module Manager):** механизм модульности, реализует BO-4.
- **FE-CORE-11 (APZ plugin universal):** реализует BO-5.
- **FE-CORE-12..13 (Observability + E2E baseline):** production-ready минимум.

**Optional-модули в v1.0 (поставляются в репозитории, но не включены в core):**

- **FE-OPT-BILLING (Stripe):** включается, если проект требует подписки. Без него core работает.
- **FE-OPT-MCP + FE-OPT-CMDBAR:** включаются вместе для AI-first-проектов. Без них core работает.
- **FE-OPT-JOBS (Trigger.dev):** включается для проектов с тяжёлыми задачами. Без него core работает.
- **FE-OPT-E2E-FULL:** подключается вместе с AI-модулями.

**Optional-модули, перенесённые на v1.1+ (не в v1.0):**

- **FE-OPT-FINOPS:** LiteLLM + FinOps UI. Перенесён: автор сознательно не включает в v1.0 — опциональная надстройка над MCP-модулем, нужна не всем проектам и не с первого дня. Будет добавлен в v1.1/v1.2, когда соберётся реальная потребность.

**Ожидаемая дата релиза:** *TBD. Зависит от загрузки автора; Schedule в приоритетах принимается как «принять» (см. 4.2).*

### 3.2 Scope of Subsequent Releases

**v1.1 — v1.x (минорные улучшения после v1.0):**

- **FE-OPT-FINOPS** — LiteLLM + FinOps UI как полноценный optional-модуль. Критерий активации: появилась реальная потребность в per-org/per-project учёте LLM-расходов хотя бы в одном проекте на базе Kit.
- Дополнительные OAuth-провайдеры (GitHub, Microsoft) сверх базовых v1.0.
- Расширение `.cursorrules`, `CLAUDE.md` и meta-commands по обратной связи первых проектов.
- Дополнительные адаптеры `BillingProvider` (например, Paddle).
- Дополнительные опциональные модули по мере появления повторяющихся потребностей (Notifications, File Storage, Search, Webhooks Outbound и т. д.).

**v2.0 (ожидается +6 мес):**

- **Локализация (i18n):** поддержка нескольких языков интерфейса.
- **Мобильное приложение (React Native / Expo):** отложено — фокус v1.0 на web.
- **SCIM / SAML (enterprise-auth):** отложено — целевая аудитория не требует.
- **Self-host документация и infra-шаблоны (Terraform / Pulumi):** отложено после валидации спроса.
- **Расширенные AI-оркестрации (LangGraph / Temporal для AI-flows):** отложено до понимания типовых потребностей.

**v3.0 и далее:**

- Marketplace модулей (vertical-specific optional-модули поверх core FASTSAAS).
- Визуальный no-code-конструктор сущностей поверх `add-entity`.
- Публичная телеметрия/бенчмарки «AI-готовности» кодовых баз.

### 3.3 Limitations and Exclusions

Следующее **НЕ входит** в scope проекта FASTSAAS:

1. **Готовая бизнес-логика конкретной вертикали** — шаблон не предлагает модели и экраны для CRM, ERP, e-commerce, marketplace и т. п.; только примеры сущностей для демонстрации.
2. **Хостинг и managed-сервис** — FASTSAAS поставляется как код; мы не размещаем чужие инсталляции.
3. **Обучение и обеспечение соответствия** (GDPR / HIPAA / SOC2) — шаблон предоставляет технические примитивы (audit log, изоляция), но compliance-сертификация — ответственность пользователей шаблона.
4. **Мобильные приложения** (iOS / Android native) — out of scope.
5. **Тяжёлый enterprise-auth** (SAML, SCIM, LDAP) — out of scope для v1.0 (см. v2.0).
6. **Интеграции с конкретными CRM/маркетинговыми инструментами** (HubSpot, Intercom и т. п.) — out of scope; оставлено на стороне пользователя.
7. **Поддержка альтернативных СУБД** кроме PostgreSQL (MySQL, SQLite в проде, MongoDB) — out of scope.
8. **Поддержка альтернативных фронтенд-фреймворков** (Remix, SvelteKit, Nuxt) — out of scope.
9. **Собственный LLM-inference / fine-tuning** — шаблон использует внешние LLM через LiteLLM, не предоставляет self-hosted inference.
10. **Локализация (i18n)** — v1.0 только en-US.

### 4.3 Deployment Considerations

**Целевая среда:**

FASTSAAS поставляется как git-репозиторий-шаблон. «Развёртывание» для нас означает:

- Локальная dev-среда: `docker-compose up` поднимает Postgres, Redis, Mailhog; `make dev` запускает backend и frontend.
- Production: предполагаются Vercel/Railway/Fly.io/AWS — шаблон содержит примеры `Dockerfile` и инструкции, но managed-deploy не автоматизируется в v1.0.

**Миграция данных:**

Не применимо — шаблон не заменяет существующую систему; пользователи шаблона стартуют с чистой БД.

**Обучение и переход:**

- `README.md` с quick-start (≤ 15 минут до работающего локального инстанса).
- `CLAUDE.md` / `AGENTS.md` — инструкции для AI-агентов по работе с кодовой базой.
- `docs/` — архитектурные заметки: Actor-Centric model, Audit Trail, MCP, FinOps, DX-конвенции.
- Эталонный screen-cast / walkthrough (опционально, v1.1).

## 5. Appendix (Приложения)

### 5.1 Глоссарий

| Термин | Определение |
|-------------------------|-----------------------------------------------------------------------------------------------------------------------------------------|
| Actor | Единица идентичности в системе, способная инициировать действия. Типы: HUMAN, AGENT. |
| Actor-Centric model | Архитектурный подход, в котором ключевой сущностью идентификации является Actor (а не User), что позволяет первоклассно моделировать AI-агентов. |
| AGENT | Actor-Type, обозначающий AI-агента, действующего от имени HUMAN (с заполненным `parent_actor_id`). |
| `parent_actor_id` | Внешний ключ на HUMAN-актора, от имени которого действует AGENT; обеспечивает accountability. |
| `intent_hash` | Детерминированный хеш «намерения» действия (формулировка пользователя + параметры), позволяющий группировать связанные операции в audit trail. |
| MCP (Model Context Protocol) | Протокол, по которому AI-агент подключается к серверу и получает доступ к набору инструментов и ресурсов. |
| LiteLLM | Прокси-gateway для унифицированного доступа к LLM-провайдерам с учётом токенов и стоимости. |
| FinOps | Практика управления и контроля облачных/LLM-расходов, их прозрачность и атрибуция. |
| Vibe Coding | Режим разработки, в котором значительную часть кодовой базы пишет и правит AI-агент; требует кодовой базы, спроектированной под AI. |
| Trigger.dev | Платформа фоновых задач с TypeScript-SDK, используется для тяжёлых операций (Import/Export) в optional-модуле FE-OPT-JOBS. |
| shadcn/ui | Набор копипаст-компонентов на Radix UI, кастомизируемых через Tailwind и CSS-переменные. |
| Meta-command | Задокументированный сценарий работы с AI-агентом (например, `add-entity`), реализуемый на уровне структуры репозитория и `.cursorrules`. |
| Module (Kit-module) | Архитектурная единица FASTSAAS: core (обязательный) или optional. Имеет собственные модели, миграции, API-роуты, UI-компоненты и конфиг; может подключаться/отключаться через Module Manager. |
| APZ plugin (universal) | Универсальная (не привязанная к FASTSAAS) версия Claude-Code-плагина с командами `/apz:req:*`, `/apz:build:*`, `/apz:ship:*`, поставляемая как часть FASTSAAS. |
